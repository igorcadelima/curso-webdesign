O que é uma função?

  • (Substantivo) Um bloco de código reusável com um nome que é designado para realizar uma tarefa em particular.
  • Algumas vezes é chamada de método.
  • alert, prompt, e typeof são exemplos de funções que usamos até agora.

Por que criar funções?

  • Reúso de código
    • Você estará fazendo a mesma coisa sempre
  • Código simples
    • Funções têm nomes que definem o que elas fazem
    • Quebra blocos complexos de código em partes menores
  • Mais fácil para fazer mudanças
    • Se foi escrito apenas uma vez, você só tem que mudar uma vez

Criando funcões

As duas maneiras mais comuns de criar funções em JS são as seguintes: declaração de função e expressão de função.

Declaração de função:

function nome(parametro1, parametro2, ..., parametroN){
  // Código a ser executado
  return;
}

Expressão de função:

var nome = function(parametro1, parametro2, ..., parametroN) {
  // Código a ser executado
  return;
}

Parâmetros são opcionais.

Criando e chamando funções - exemplo

function exibirAlerta() {
  alert("Mensagem de alerta!");
  return;
}

exibirAlerta();

Criando e chamando funções - exemplo 2

function escreverNomes() {
  var nomes = ["Guinho", "Zezinho", "Luisinho"];
  for (var i = 0; i < 3; i++) {
    document.write(nomes[i] + "</br>");
  }
  return;
}

escreverNomes();

Dinamicidade nas funções

Em nossos exemplos as funções faziam apenas uma coisa — algumas vezes exatamente o que queríamos.

No entanto, algumas vezes precisamos de um pouco de flexibilidade

  • Criar messagens personalizadas para serem exibidas
  • Prover dois números para fazer um cálculo
  • Validar campos de formulários, e opcionalmente exibir mensagens de aviso na tela

Parâmetros

Para criar funções que aceitam dados você pode usar parâmetros:

  • Um parâmetro é a definição de um dado que uma função pode receber;
  • Dentro de funções, parâmetros são como variáveis.
function exibirAlerta(mensagem) { // Mensagem é um parâmetro
  alert(mensagem);
  return;
}

exibirAlerta("Olá, mundo!");

Argumentos

Para chamar funções que aceitam dados e passar dados você pode usar argumentos:

  • Um argumento é um valor real passado para uma função em tempo de execução do programa.
  • Vocês já usaram argumentos antes: alert("Olá, mundo!");
function exibirAlerta(mensagem) { // Mensagem é um parâmetro
  alert(mensagem);
  return;
}

exibirAlerta("Olá, mundo!"); // "Olá, mundo!" é um argumento

Parâmetros vs Argumentos

Não confundam parâmetros com argumentos:

  • Parâmetro é apenas a definição do dado usada para construir a função;
  • Argumento é o valor que será, de fato, passado para a função.

Múltiplos parâmetros

Como podemos criar uma função que pode receber vários argumentos?

  • Simplesmente adicione os parâmetros separando-os com vírgula.
function exibirMensagem(cumprimento, nome) {  // Parâmetros
  var mensagem = cumprimento + ", " + nome;
  alert("mensagem");
  return;
}

exibirMensagem("Oi", "Igor"); // Argumentos

Passando valores para funções - exemplo 1

function exibirNomes(nomes) {
  for (var i = 0; i < nomes.length; i++) {
    document.write(nomes[i] + "</br>");
  }
  return;
}

var lista = ["Guinho", "Zezinho", "Luisinho"];
var novoNome = prompt("Nome do próximo convidado:");
lista.push(novoNome);
exibirNomes(lista);

Escopo de variável

Variáveis locais:

  • São variáveis criadas dentro do contexto de uma função e só funciona dentro da mesma.

Variáveis globais:

  • Devem ser declarada no contexto principal do programa para que possa ser acessada por todas as funções.

Retorno de dados

Especifique o valor ou dado que deseja retornar após a palavra-chave return.

Você pode reusar nomes em funções diferentes:

function getMensagem(nome) {
  var mensagem = "Olá, " + nome;
  return mensagem;
}
function exibirAlerta(mensagem) {
  alert(mensagem);
  return;
}
var saida = getMensagem("Igor");
exibirAlerta(saida);

Eventos

Eventos

Pode-se dizer que evento é tudo que um usuário realiza quando está visitando uma página web. Ex.: mover ou clicar com o mouse sobre um objeto de texto, de imagem, ou de formulário.

Outros exemplos:

  • Carregamento de uma página;
  • Passagem do mouse sobre algum elemento HTML;
  • Seleção de um campo de entrada em um formulário HTML;
  • Submissão de um formulário HTML;
  • Pressionamento de tecla;
  • e muitos outros.

Captura de eventos

Usando declaração inline e incorporada de código JS para capturar eventos:

<head>
  <meta charset="UTF-8">
  <title>Exemplo</title>
  <script>
    function acao() {
      alert("Opa! Você acabou de clicar no campo de email.");
    }
  </script>
</head>
<body>
  Email: <input id="email" name="email" type="email" onClick="acao();">
</body>

Captura de eventos

Usando declaração externa de código JS para capturar eventos:

<head>
  <meta charset="UTF-8">
  <title>Exemplo</title>
  <script>
    function acao() {
      alert("Opa! Você acabou de clicar no campo de email.");
    }
    document.getElementById("email").onclick = acao;
  </script>
</head>
<body>
  Email: <input id="email" name="email" type="email">
</body>

Tipos de eventos

Esses serão os eventos vistos nessa aula:

Evento Descrição
onLoad Acontece quando um objeto é carregado.
onClick Acontece quando um objeto é clicado.
onMouseOver Acontece quando o cursor é colocado sobre um objeto.
onMouseOut Acontece quando o curso deixade estar sobre um objeto.
onSelect Acontece quando um campo de formulário é selecionado.
onFocus Acontece quando um elemento de formulário recebe foco.
onBlur Acontece quando um elemento de formulário perde foco.
onChange Acontece quando um objeto é modificado.
onSubmit Acontece quando a ação de submissão de um formulário é iniciada.

onSubmit

...
  <script>
    function submissao() {
      var login = document.getElementById('login').value;
      var senha = document.getElementById('senha').value;
      if (login == "") {
        alert("Digite seu login.");
      }
      if (senha == "") {
        alert("Digite sua senha.");
      }
    }
  </script>
...

    </form>
  </body>
